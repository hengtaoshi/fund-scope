#### 1. 沪深300 对比曲线不可见 — 纵轴量级不匹配

**位置**：`index.html` 第 1724-1766 行（`renderLineChart` 函数）

**现象**：

```javascript
const portfolio = [118000, 119200, 121500, ...]; // 绝对值：11.8万 ~ 12.9万
const hs300     = [100, 99.5, 101.2, ...];       // 归一化值：99.5 ~ 104
```

两组数据共用单一 y 轴，portfolio 在 12 万量级，hs300 在 100 量级，相差 1000 倍。hs300 曲线会被压成一条贴地直线，用户完全看不到"跑赢基准"的效果。

**看法**：这是 demo 中"累计收益走势"图表的核心卖点——展示组合相对沪深300 的超额收益。现在图表形同虚设，等于核心功能缺失。

**建议**：二选一：
- 方案 A：将 portfolio 也归一化为 100 基准，单轴对比涨跌幅（推荐，直观）
- 方案 B：保持绝对值，给 hs300 配置独立右侧纵轴（`yAxisID: 'y1'`）

#### 2. 持仓明细汇总与总资产严重不一致

**现象**：四个持仓市值之和为 **¥28,695**，但页面 4 处显示总资产为 **¥128,640**，差距 **4.5 倍**。

| 基金 | 份额 | 当前净值 | 市值 | 逐笔验证 |
|------|------|---------|------|---------|
| 易方达蓝筹精选 | 5,000 | 1.6830 | ¥8,415 | 5000 × 1.6830 = 8,415 ✓ |
| 中欧医疗健康C | 3,200 | 1.1850 | ¥3,792 | 3200 × 1.1850 = 3,792 ✓ |
| 富国天惠成长 | 2,800 | 2.3850 | ¥6,678 | 2800 × 2.3850 = 6,678 ✓ |
| 招商中证白酒 | 10,000 | 0.9810 | ¥9,810 | 10000 × 0.9810 = 9,810 ✓ |
| **逐笔合计** | | | **¥28,695** | |
| **页面总资产** | | | **¥128,640** | ❌ 不匹配 |

此外，"总盈利 +¥10,093"同样对不上——各基金盈亏分别为 +815 / -144 / +798 / +310，实际合计 **+¥1,779**。

饼图数据用 ¥28,695 分布绘制，走势图用 ¥128,640 系列，两套数据平行存在、互不校验。

**看法**：这是 demo 最致命的问题。对金融类产品而言，数据准确性是信任的底线。如果给潜在用户或投资人演示，对方第一时间用计算器加总发现对不上，整个 demo 就失去说服力。

**建议**：
- 先选一条"真相基线"。如果想展示十万级资产规模，把每只基金的持有份额等比放大 4.5 倍
- 在 JS 中由一个数据源统一派生所有展示值，而非 HTML 写一套、JS 写一套

### 较高优先级（安全 / 架构隐患）

#### 3. SMTP 凭证在客户端明文处理

**位置**：`index.html` 第 1179-1209 行（邮箱配置弹窗）

前端直接收集 SMTP 服务器、端口、邮箱地址、**授权码**四个字段。虽然当前仅做 mock 并未真实发送，但代码结构和提示文案（"QQ邮箱需开启SMTP后获取专用授权码"）暗示了"接入真实 SMTP 即可使用"的设计意图。

**看法**：如果在生产环境按当前架构落地，SMTP 授权码会在浏览器端明文暴露，攻击者可以直接用此凭证批量发送垃圾/钓鱼邮件。这是典型的"不该放在前端的东西"。

**建议**：
- 前端只收集邮箱地址和通知偏好，将发送逻辑移到后端 API
- 后端使用环境变量或密钥管理服务存储 SMTP 凭证
- 在前端表单旁添加明确说明："当前为演示模式，邮件功能需配合后端使用"

#### 4. `innerHTML` 字符串拼接 — 潜在 XSS 注入点

**位置**：`index.html` 第 1546-1586 行（`openFundDetail` 函数）

```javascript
document.getElementById('fdComment').innerHTML = '<i ...></i> ' + f.comment;
document.getElementById('fdChange').innerHTML = f.change;
document.getElementById('fdR1m').innerHTML = f.r1m;
// ... 共十余处类似模式
```

当前 `fundData` 是硬编码常量所以安全，但如果后续改为对接基金 API 或允许用户自定义备注，攻击者可在净值描述/基金经理等字段注入 `<script>` 或 `<img onerror>`。

**看法**：这是"现在没问题但未来很容易踩坑"的模式。demo 阶段的架构选择往往会被后续开发直接沿用。

**建议**：
- 对于纯文本字段，统一使用 `textContent` 替代 `innerHTML`
- 如果确实需要富文本（如涨跌颜色），使用 `classList` 控制样式而非内嵌 HTML 标签
- 或者将 HTML 标签从数据中剥离，数据层只存纯值，渲染层负责加标签

#### 5. 弹窗组件与页面容器耦合

**位置**：`index.html` 第 1179 行（`emailConfigModal` 嵌套在 `page-settings` 内）

```html
<div class="page" id="page-settings">   <!-- display: none 当不在设置页时 -->
    ...
    <div class="modal-overlay" id="emailConfigModal">  <!-- 被父级 display:none 压制 -->
```

当前因为打开该弹窗的唯一入口（通知 toggle）也在设置页内，所以"恰好能用"。但弹窗使用 `position: fixed` 的设计意图是全局覆盖，实际却受限于父容器的 `display` 状态。

**看法**：这是前端架构中的"偶然正确"——不是设计使然，而是运气使然。一旦产品需求要求在其他页面也能触发邮箱配置，bug 就会暴露。

**建议**：将所有 modal 移到 `</body>` 之前的最外层，与页面容器同级。当前 `emailModal` 和 `fundDetailModal` 的放置就是正确的，`emailConfigModal` 应参照它们处理。

### 中等优先级（健壮性 / 可维护性）

#### 6. 定投计算器无输入校验

**位置**：`index.html` 第 1807-1828 行（`calcDCA` 函数）

用户可以输入负数金额、负收益率、0 年限，得出无意义结果且无提示。

**建议**：
- 金额和年限限制 `min="1"`
- 收益率允许负数但给出明确的风险提示
- 极端值（如月投 1 亿）增加简单的上限校验或警告

#### 7. `setTimeout` 等待 DOM 渲染存在竞态

**位置**：`index.html` 第 1678 行

```javascript
setTimeout(() => { renderCharts(); }, 100);
```

100ms 在慢设备或页面复杂时可能不够。**建议**：使用 `requestAnimationFrame` 或在 CSS transition 的 `transitionend` 事件中触发。

#### 8. Toggle 开关不可通过键盘操作

**位置**：`index.html` 第 326-344 行（CSS）+ 第 1831-1843 行（JS）

所有 toggle 用 `<div>` 实现，缺少 `role="switch"`、`aria-checked`、`tabindex="0"`，屏幕阅读器和键盘用户完全无法感知和操作。

**建议**：用 `<button role="switch" aria-checked="true|false">` 或隐藏的 `<input type="checkbox">` 重构。

### 较低优先级（细节打磨）

#### 9. HTML 与 JS 数据双维护

智能预测页的四张基金卡片（第 825-1078 行，约 250 行纯 HTML）与 `fundData` 数组（第 1420-1537 行）数据完全重复。修改评分或信号标签需要改两份，必然出现不同步。

**建议**：预测卡片改为从 `fundData` 循环渲染生成，消除硬编码 HTML。

#### 10. 日期硬编码

```html
<!-- line 493 -->
<span class="date">2025年7月14日</span>
```

**建议**：改为 `new Date().toLocaleDateString('zh-CN', {...})` 动态生成。

#### 11. Chart.js CDN 无降级处理

如果 `cdn.jsdelivr.net` 不可达，`new Chart(...)` 会抛出 `ReferenceError`，整个脚本中断，页面所有交互失效。

**建议**：
```javascript
if (typeof Chart === 'undefined') {
    console.warn('Chart.js 加载失败，图表功能不可用');
    // 在图表容器显示友好提示
}
```
