## 批处理一：数据可信度（P0）

> 影响 demo 核心说服力，必须先修再进 Phase 2

### 问题 2 ⚠️ 持仓金额不一致

**根因**：四处数据各自写死，无统一数据源。

| 位置 | 金额 | 来源 |
|------|------|------|
| 持仓明细表 4 只基金合计 | ¥28,695 | HTML 硬编码 |
| 总资产卡片 | ¥128,640 | HTML 硬编码 |
| 投资总成本 | ¥118,547 | HTML 硬编码 |
| 总盈利 | +¥10,093 | HTML 硬编码 |
| 饼图数据 | [8415, 3792, 6678, 9810] | JS 硬编码 |

**修复方案**：统一数据源。以持仓明细表为真相源
- 将 4 只基金的 **份额 × 净值** 作为基准值
- 总资产 = 各基金市值之和
- 总成本 = 各基金份额 × 成本净值之和
- 总盈利 = 总市值 - 总成本
- 饼图数据、走势图数据都从同一来源派生
- 修正后的数据：每只基金份额放大到接近 15-20 万总资产级别

### 问题 1 ⚠️ 沪深300 对比曲线不可见

**根因**：portfolio（12 万量级）和 hs300（100 量级）共用单 y 轴。

**修复方案**：采用方案 A — 将 portfolio 也归一化为 100 基准
```
基准日 2025-01-01 组合净值 = 100
之后每月 = 当月组合市值 / 基准日市值 × 100
hs300 同理
```

这样两组数据都在 100 附近波动，涨跌幅对比一目了然。

## 批处理二：安全与架构（P1）

> 不阻塞 demo 展示，但决定了 Phase 2 的代码质量

### 问题 3 SMTP 凭证前端明文

**当前**：前端直接收集授权码
**修复**：
- 前端只保留「接收邮箱」字段
- 弹窗说明文案改为"当前为演示模式，邮件功能需配合后端"
- SMTP 配置（服务器/端口/授权码）仅在后端通过环境变量管理
- 保留邮箱字段仅用于显示"报告将发送至 xxx"

### 问题 4 innerHTML → textContent

**当前**：13 处 `innerHTML` 拼接含用户数据
**修复**：
- 纯文本字段（基金名称、评论、建议）→ `textContent`
- 收益数字（可能为负/正）→ 用 `<span class="text-up/down">` 包裹
- 保留 `innerHTML` 仅用于确实需要 HTML 渲染的结构（如持仓列表）

### 问题 5 弹窗移出 page 容器

**当前**：`emailConfigModal` 嵌套在 `#page-settings` 内
**修复**：将 `emailConfigModal` 移到 `</div>`（主内容结尾）之后，与 `emailModal`、`fundDetailModal` 同级

### 问题 9 消除 HTML/JS 数据双维护

**当前**：预测页 4 张卡片是硬编码 HTML + JS 中的 `fundData` 重复
**修复**：预测页卡片改为 JS 循环渲染，HTML 中只保留容器骨架
